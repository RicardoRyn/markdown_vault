# 在服务器上通过SSH逆向隧道连接外网

## 第一步

在台式机上执行`server_proxy`。

这是个自定义的别名，实际上执行的是`ssh -v -N -R <自定义的一个端口>:localhost:<clash端口> <用户名>@<服务器地址>`。

- **`-R xxx`**（核心）：这是 **Remote Forwarding**。
  它告诉服务器 B：“监听你自己的 `<自定义的端口>` 端口，任何发往你 `<自定义的端口>` 的流量，请通过 SSH 隧道转发到我（台式机 A）的 `<clash端口>` 端口”。
- **`-N`**：只建立连接，不打开远程 Shell 窗口（纯隧道模式）。
- **`-v`**：Verbose 模式，打印调试日志（所以你会看到一堆输出，方便检查隧道通没通）。

## 第二步

另开一个终端通过ssh连接入服务器，在服务器上执行`rjxproxy`。

这也是个自定义的别名，实际上执行的是`export ALL_PROXY=socks5h://127.0.0.1:<刚刚自定义的端口>`。

## 总结

- **本地台式机 A**：运行着 Clash，监听在 `<clash端口>` 端口（这是你本地的“梯子”入口）。
- **服务器 B**：本身没梯子，但它通过 SSH 隧道把自己的 `<自定义的一个端口>` 端口“借”给了 A。
- **隧道连接**：当你执行 `server_proxy` 时，A 和 B 之间建立了一条秘密通道。

这套配置**不需要在服务器上保存任何 VPN 账号信息**，只要你的台式机在线，服务器就能借用你的网络
